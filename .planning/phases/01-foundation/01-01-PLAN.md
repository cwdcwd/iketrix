4. Create `prisma/schema.prisma` with these models:
   ```prisma
   generator client {
     provider = "prisma-client"
     output   = "../generated/prisma"
   }

   datasource db {
     provider = "postgresql"
   }

   model User {
     id              String    @id @default(cuid())
     clerkId         String    @unique
     email           String    @unique
     name            String?
     createdAt       DateTime  @default(now())
     updatedAt       DateTime  @updatedAt
     tasks           Task[]
     sources         Source[]
     overrides       ClassificationOverride[]
   }

   model Source {
     id              String    @id @default(cuid())
     type            String    // "github", "jira", "linear", etc.
     name            String    // e.g., "owner/repo"
     accessToken     String    // encrypted OAuth token
     userId          String
     user            User      @relation(fields: [userId], references: [id], onDelete: Cascade)
     tasks           Task[]
     createdAt       DateTime  @default(now())
     updatedAt       DateTime  @updatedAt

     @@unique([userId, type, name])
   }

   model Task {
     id              String    @id @default(cuid())
     title           String
     description     String?
     sourceId        String?
     source          Source?   @relation(fields: [sourceId], references: [id], onDelete: SetNull)
     externalId      String?   // GitHub issue number, etc.
     externalUrl     String?
     userId          String
     user            User      @relation(fields: [userId], references: [id], onDelete: Cascade)
     quadrant        String?   // "do", "schedule", "delegate", "delete"
     status          String    @default("active") // "active", "delegated", "completed", "deleted"
     classification  TaskClassification?
     overrides       ClassificationOverride[]
     delegatedTo     String?   // email or identifier of delegatee
     delegatedAt     DateTime?
     createdAt       DateTime  @default(now())
     updatedAt       DateTime  @updatedAt

     @@unique([sourceId, externalId])
   }

   model TaskClassification {
     id              String    @id @default(cuid())
     taskId          String    @unique
     task            Task      @relation(fields: [taskId], references: [id], onDelete: Cascade)
     quadrant        String    // "do", "schedule", "delegate", "delete"
     reasoning       String    // LLM explanation
     confidence      Float     // 0-1 score
     model           String    // which LLM model was used
     createdAt       DateTime  @default(now())
   }

   model ClassificationOverride {
     id              String    @id @default(cuid())
     taskId          String
     task            Task      @relation(fields: [taskId], references: [id], onDelete: Cascade)
     userId          String
     user            User      @relation(fields: [userId], references: [id], onDelete: Cascade)
     fromQuadrant    String
     toQuadrant      String
     createdAt       DateTime  @default(now())
   }
   ```

5. Create `src/lib/prisma.ts` — Prisma client singleton with PrismaPg adapter:
   ```typescript
   import "dotenv/config";
   import { PrismaPg } from "@prisma/adapter-pg";
   import { PrismaClient } from "../../generated/prisma/client";

   const globalForPrisma = globalThis as unknown as { prisma: PrismaClient };

   const connectionString = process.env.DATABASE_URL!;
   const adapter = new PrismaPg({ connectionString });

   export const prisma = globalForPrisma.prisma || new PrismaClient({ adapter });

   if (process.env.NODE_ENV !== "production") globalForPrisma.prisma = prisma;
   ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API | Browser requests to Next.js API routes |
| API → Database | Server-side Prisma queries to Postgres |
| API → Clerk | Auth verification via Clerk SDK |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | Auth | mitigate | Clerk handles all auth — clerkMiddleware() validates sessions on every request |
| T-01-02 | Information Disclosure | .env | mitigate | .env in .gitignore, .env.example has placeholder values only |
| T-01-03 | Information Disclosure | accessToken in Source | mitigate | Store encrypted OAuth tokens; never expose in API responses |
| T-01-04 | Elevation of Privilege | API routes | mitigate | All data-modifying routes require authenticated Clerk session; userId from session, never from client |
</threat_model>
